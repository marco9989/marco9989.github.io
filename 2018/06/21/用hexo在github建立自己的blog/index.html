用Hexo在Github建立自己的Blog

发表于 2018-06-21 |

首先，本人非技术码农，完全不懂代码，但是又非常想在茫茫互联网开垦一个属于自己的小天地。

第一次知道HEXO是在科技聚变网站，其实这是一档由两个海外学子所办的同名播客的附属网站，我是因为听这档播客节目才有会看到这个网站，可能是看惯了公众号或者微博、twitter的缘故吧。第一次看到感觉网页很清爽干净，用现在的流行说法，有点佛系韵昧，于是写邮件问了这个网站的后台所用技术，于是就听到了HEXO这个程序；

第二次看到HEXO是在李笑来的网站，原来读李笑来的公众号，觉得是一个写字的人，没想到他其实很懂技术，还会写代码，还写了许多关于写代码的教程，其中有一篇叫“使用Hexo为自己在Github上建一个静态Blog站点”，自己马上产生了兴趣，跟着完成了这个Blog站点的搭建。

做任何事情开头都是比较难的，有时候就是依葫芦画瓢也很难，因为对于代码的完全陌生，还有对unix命令的无知，通过简单的复制命令，然后按个回车，顺利还好，如果出现“error”字样便手足无措了，于是只能借助google，根据错误信息查询解决方案，一来二去，有些错误试了挺多遍，终于还是得到了解决方案。

今天写这篇文章的目的就是想通过自己搭建这个Blog站点的尝试经历，结合网上各位大神的经验分享，写一篇教程汇总一下，给和我一样的代码小白，也为了以后自己所需吧。

下面我们说一下以后要用到的几个常用命令:

hexo new "新文章" #创建一篇新文章，引号里的名字随你喜欢，成功以后可以在本地文件夹 name123.github.io/source/_posts 里找到一个名为 新文章.md 的文件，用atom编辑就行了。

hexo generate #这是生成新网页的命令，每当编辑完或者修改完以后发布之前先用这个命令编译一下。

hexo server #这是本地服务命令，运行这个命令后，在浏览器里输入localhost:4000可以查看自己本地的网页效果

hexo deploy #这是发布命令，使用这个命令就可以把刚才编译完以后的新文件发布到github了。
